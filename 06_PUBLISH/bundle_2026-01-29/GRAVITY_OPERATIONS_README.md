## Mandatory Execution Rhythm

Every task follows this order:

1. VERIFY  
   Confirm current state. No assumptions.

2. PATCH  
   Apply the smallest change possible.

3. TEST  
   Prove the change worked.

4. LOG  
   Record what happened and why.

5. PUBLISH  
   Package artifacts into a dated bundle.

If any step is skipped, the task is not complete.

---

## Logging Standard

All work must result in a log entry under:

C:\GRAVITY\04_LOGS

Logs must include:
- phase labels
- evidence of verification
- summary of changes
- final status

Logs are factual, not emotional.
